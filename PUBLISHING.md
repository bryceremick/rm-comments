# Publishing runbook

What YOU need to do, per surface. Everything not listed here is automated
(CI workflows, plugin marketplace, Zed integration need no action).

## One-time setup

### 1. crates.io (`cargo install rm-comments`)

1. Go to <https://crates.io> → **Log in with GitHub**.
2. **Account Settings → API Tokens → New Token.**
   - Name: anything (`publish-laptop`)
   - Scopes: `publish-new` + `publish-update`
   - Expiry: your call (you'll regenerate when it lapses)
3. In a terminal, from the repo root:

   ```sh
   cargo login          # paste the token when prompted
   cargo publish
   ```

4. Verify: <https://crates.io/crates/rm-comments> exists and
   `cargo install rm-comments` works in a clean shell.
5. docs.rs builds automatically within ~10 minutes of publishing — nothing to do.

### 2. GitHub Releases (prebuilt binaries)

Nothing to set up. The workflow (`.github/workflows/release.yml`) uses the automatic
`GITHUB_TOKEN` — no secrets needed. Pushing a `v*` tag triggers it (see per-release
checklist). Only check: repo **Settings → Actions** must allow workflows (default: allowed).

### 3. Homebrew tap (`brew install bryceremick/tap/rm-comments`)

The tap repo is `bryceremick/homebrew-tap`; the formula is regenerated by
`scripts/update-brew-formula.sh` after each release. One-time: nothing, as long as your
`gh` auth can push to that repo (it's your own repo, so it can).

### 4. cargo-binstall (`cargo binstall rm-comments`)

Nothing. The `[package.metadata.binstall]` section in `Cargo.toml` points binstall at the
GitHub Release artifacts. Works as soon as crates.io publish + a release exist for the
same version.

### 5. Claude Code plugin

Nothing to publish — the GitHub repo IS the marketplace. Users run:

```
/plugin marketplace add bryceremick/rm-comments
/plugin install rm-comments@rm-comments
```

For your own machine, run those two commands in Claude Code once.
Optional reach: submit the marketplace to community aggregator lists — search GitHub for
"awesome-claude-code-plugins" and open a PR if you care.

### 6. Cleanup (leftover from the rename)

The old repo still exists with the pre-rename history. Delete it:

```sh
gh auth refresh -h github.com -s delete_repo
gh repo delete bryceremick/zed-strip-comments --yes
```

## Per-release checklist

1. Bump the version in **`Cargo.toml`** and **`.claude-plugin/plugin.json`**
   (keep them in sync), commit, push.
2. Tag and push the tag — this builds all binaries and creates the GitHub Release:

   ```sh
   git tag vX.Y.Z
   git push && git push --tags
   ```

3. Wait for the `release` workflow to go green
   (`gh run watch` or the Actions tab).
4. Publish to crates.io:

   ```sh
   cargo publish
   ```

5. Update the Homebrew formula (downloads artifacts, computes SHAs, pushes to the tap):

   ```sh
   ./scripts/update-brew-formula.sh vX.Y.Z
   ```

6. Verify the surfaces:

   ```sh
   cargo install rm-comments --force && rm-comments --help
   brew update && brew upgrade rm-comments   # or brew install bryceremick/tap/rm-comments
   cargo binstall rm-comments --force
   ```

Plugin and Zed users need no action; plugin marketplaces update from the repo,
and Zed tasks call whatever binary is installed.

## Auth summary

| Surface | Credential | Where it lives |
|---|---|---|
| crates.io | API token via `cargo login` | `~/.cargo/credentials.toml` |
| GitHub Releases | automatic `GITHUB_TOKEN` | nothing on your side |
| Homebrew tap | your existing `gh` auth (push to own repo) | `gh auth status` |
| binstall / docs.rs / plugin / Zed | none | — |
